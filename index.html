1. Current State Summary
Critical Issue: လက်ရှိတွင် Authenticated ဖြစ်သော User တိုင်းသည် အရာအားလုံးကို လုပ်ဆောင်နိုင်နေသည်။ Role ခွဲခြားထားခြင်း မရှိသေးပါ။
Component Current Behavior
User Model Role column မရှိပါ။ name, email, password သာရှိသည်။
API Routes Middleware မရှိပါ။ Public ဖြစ်နေသည်။
Gates / Policies မရှိသေးပါ။ FormRequest အားလုံး true ပြန်နေသည်။
2. Security Gaps & Risks
API Vulnerability

/api/v1/routes (POST/DELETE) သည် မည်သူမဆို (Public) ခေါ်ယူသုံးစွဲနိုင်နေပြီး Route များကို ဖျက်ဆီးနိုင်သည်။

Editor Bypass

Editor သည် UI တွင် Button ဖျောက်ထားသော်လည်း URL ကို တိုက်ရိုက်ခေါ်ခြင်း (curl) ဖြင့် Shop များအား Mutate လုပ်နိုင်သည်။

4. Six-Phase Implementation Plan
Phase 1: Foundation

Users table တွင် Role ထည့်ခြင်းနှင့် AuthServiceProvider ၌ Gates များ သတ်မှတ်ခြင်း။

Phase 2: Controller Protection

ShopController နှင့် RouteController တို့တွင် Gate::authorize() ထည့်သွင်းခြင်း။

Phase 3: API Security

API Route များကို auth:sanctum အောက်သို့ ရွှေ့ပြီး Authorization စစ်ဆေးခြင်း။

Phase 4: Blade UI Tweaks

@can directive ကိုသုံး၍ Role အလိုက် ခလုတ်များကို ဖျောက်/ပြ လုပ်ဆောင်ခြင်း။

Phase 5: Redirect Logic

Login ဝင်ပြီးနောက် Role အလိုက် သက်ဆိုင်ရာ Dashboard များသို့ ပို့ဆောင်ပေးခြင်း။

Phase 6: Testing

Editor သည် Mutate လုပ်၍မရကြောင်းနှင့် API သည် လုံခြုံကြောင်း စစ်ဆေးခြင်း။
